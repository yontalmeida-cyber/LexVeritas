🔒 Dados & Segurança Jurídica

O que acontece ao seu documento jurídico
quando usa Inteligência Artificial?

A diferença que ninguém explica — e que pode mudar tudo para advogados, magistrados e académicos que trabalham com documentos sensíveis.

Tony de Almeida da Costa Branco · FDUL · Fundador do LexVeritas 18 de Maio de 2026

Um advogado submete um acórdão ao ChatGPT. Recebe uma análise detalhada. Pensa: "que ferramenta útil." Mas em algum momento desse processo, o texto do processo — com os dados das partes, com a estratégia processual implícita, com a fundamentação — ficou guardado nos servidores de uma empresa americana. Pode ter sido visto por um colaborador. Pode vir a ser usado para treinar o próximo modelo de linguagem.

Não é ficção científica. É o modelo de negócio padrão das ferramentas de IA de uso geral. E as implicações para o profissional jurídico são sérias.

A questão que ninguém faz

Quando os profissionais jurídicos começaram a usar IA para analisar documentos, a conversa centrou-se quase sempre na qualidade da análise: "É precisa? É fiável? Substitui o advogado?" São perguntas legítimas. Mas há uma questão anterior, mais fundamental, que raramente é feita:

A questão essencial

O que acontece exactamente ao meu documento depois de o submeter?

A resposta varia radicalmente consoante a ferramenta utilizada. E as implicações jurídicas, deontológicas e regulatórias dessa variação são enormes.

O que acontece numa IA de uso geral

Ferramentas como ChatGPT (OpenAI), Gemini (Google), Copilot (Microsoft) ou a interface web do Claude.ai têm algo em comum: são serviços de consumo, desenhados para o grande público. Os seus termos de serviço reflectem isso.

Quando submete um documento nessas ferramentas — salvo configurações específicas de contas enterprise — acontece tipicamente o seguinte:

  • O texto é enviado para servidores da empresa, geralmente nos EUA
  • É processado pelo modelo de linguagem
  • Pode ser guardado para fins de segurança, melhoria do serviço ou treino de modelos futuros
  • Pode ser revisto por colaboradores humanos da empresa
  • A transferência de dados para os EUA pode não ter base legal clara ao abrigo do RGPD
O problema jurídico

Para um advogado, magistrado ou investigador, submeter dados de clientes, processos ou documentos sensíveis a um serviço com este modelo pode constituir violação do sigilo profissional, do RGPD e, em casos extremos, do segredo de justiça.

O que acontece no LexVeritas

O LexVeritas foi desenhado de raiz para o contexto jurídico português, com uma arquitectura que responde directamente a estas preocupações. Eis o que acontece passo a passo:

01

Extracção local no browser

Quando carrega um PDF, o texto é extraído no seu próprio browser — pelo motor pdf.js, que corre localmente no seu dispositivo. O ficheiro PDF nunca é enviado para o servidor. Apenas o texto extraído é transmitido, e apenas a parte relevante para a análise.

02

Transmissão encriptada

O texto extraído é transmitido ao servidor via HTTPS — encriptação ponto a ponto, o mesmo protocolo que protege as transacções bancárias online.

03

API Anthropic — condições contratuais distintas

O LexVeritas não usa a interface web do Claude.ai — usa a API da Anthropic, sujeita a condições contratuais radicalmente diferentes. Por contrato, a Anthropic não treina modelos de IA com dados submetidos via API. Os dados são processados em memória isolada e descartados imediatamente após a resposta. Nenhum colaborador tem acesso ao conteúdo dos documentos.

04

Apenas o resultado é guardado

O que fica no histórico é o resultado da análise: o veredicto, os indicadores, a fundamentação. O texto do documento nunca é guardado pelo LexVeritas. Está no seu dispositivo — e só lá.

A distinção crítica

Usar o Claude.ai na web não tem as mesmas garantias que usar a API da Anthropic. O LexVeritas usa exclusivamente a API — com as garantias contratuais correspondentes.

A responsabilidade que permanece

Seria desonesto concluir sem deixar claro: mesmo com estas garantias técnicas e contratuais, a responsabilidade do utilizador não desaparece. O LexVeritas recomenda e, nos seus Termos de Serviço, exige:

  • Não submeter processos em segredo de justiça
  • Avaliar a existência de base legal RGPD para dados pessoais de terceiros nos documentos
  • Respeitar o sigilo profissional aplicável à sua profissão
  • Anonimizar os documentos sempre que possível antes de submeter

A tecnologia pode oferecer garantias contratuais e arquitectónicas. A deontologia é sempre do profissional.

Conclusão

A questão não é se usar IA no trabalho jurídico. É qual ferramenta usar, com que garantias, e com que consciência das implicações.

Ferramentas de uso geral foram desenhadas para o consumidor médio — não para o advogado que submete dados protegidos pelo sigilo profissional, nem para o magistrado que trabalha com processos sensíveis. O LexVeritas foi desenhado precisamente para esse contexto.
